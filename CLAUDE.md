# peri_rept_down - Claude Code 指令

## 项目概述

A股上市公司财务数据多源清洗流水线。从Sina(AKShare)、RDS(cninfo)、PDF年报等数据源获取财务数据，通过规则引擎清洗为统一的Tidy Data。

## 关键规则

### 1. 指令执行规则

**当发现按照用户的具体指令无法完成任务时，必须向用户汇报限制并询问处理方式，不得擅自切换方案。**

例如：用户要求"通过Sina渠道下载间接法CF数据"，但Sina渠道不提供该数据时，应报告"Sina渠道不提供间接法CF数据，现有替代方案是X/Y/Z，请问如何处理？"，而不是自行切换到同花顺(THS)接口。

### 2. 数据渠道结论（2026-06-10）

本项目有三个数据渠道：

| 渠道 | 数据范围 | 优点 | 缺点 |
|------|----------|------|------|
| RDS (cninfo) | 仅2022年之前 | 数据质量高，可作质量校验标尺 | 仅有历史数据 |
| 上市公司定期报告 (PDF) | 理论上完全准确 | 权威性最高 | ①提取规则导致错漏；②退市公司可能无法下载历史财报 |
| akshare（主力渠道） | 覆盖全量 | 免费、国内主要数据源、未来主力渠道 | 需验证子渠道质量 |

akshare 已确认可用的子渠道：
- **sina（新浪财经）**：财务数据满足分析需要，**但缺少间接法现金流量表**
- **EM（东方财富）**：年报数据匹配率 96.9%，但季报匹配率极差（6-27%），不推荐作为主力渠道

间接法现金流量表获取策略（2026-06-10 更新）：
- Sina 和 EM **均不提供**间接法 CF
- THS（同花顺）接口 `stock_financial_cash_ths` 提供间接法 CF，但**精度不足（以亿为单位）**，不推荐
- **目前唯一可行方案：从 PDF 年报中提取**
- 未来不排除找到其他数据源

**总结**：
- 历史校验渠道：RDS（2022年前）+ 定期报告PDF
- 主力获取渠道：akshare-Sina（直接法 CF 可靠）
- 间接法 CF：PDF 年报提取（或 RDS 2022 前数据）
- EM 渠道：不推荐（仅年报质量可接受，但不如 Sina 稳定）

### 3. 数据源优先级

| 优先级 | 数据源 | 说明 |
|--------|--------|------|
| 1 | RDS (cninfo) | 最可靠，与PDF年报完全一致，2022年前 |
| 2 | Sina (AKShare) | 直接法CF可靠，主力数据源 |
| 3 | PDF 年报 | 权威性最高，间接法CF的唯一可行来源 |
| 4 | 东方财富 (EM) | 年报匹配率96.9%，但季报极差，不推荐 |
| 5 | THS (同花顺) | 精度不足（以亿为单位），不推荐 |

### 4. 数据精度要求

- **必须**精确到元，小数点后2位（分）
- **禁止**使用以"亿"为单位的数据源（如THS `stock_financial_cash_ths`）
- 数据来源**必须**在CSV中标注`source`列

### 4. 文件命名规则

- 下载脚本：`scripts/download_{data_source}_{statement_type}.py`
- 数据输出：`data/exports_v2/{statement_type}/{stock_code}.csv`
- 对比报告：`docs/audit/YYYY-MM-DD-{topic}.md`

### 5. Git 提交规则

- 每次提交必须有清晰的commit message
- 数据文件和脚本文件分开提交
- 规则变更必须附带测试验证
