**创建日期**: 2026-06-08
**项目**: peri_rept_down (A股财务数据多源清洗流水线)
**范围**: 全量 — 所有 `rules/*.yaml` + `data/exports_v2/` + 47,966 全量基线对比
**焦点**: 规则与数据质量专项
**交付物**: 量化指标报告 + 缺口清单
**形式**: `docs/audit/2026-06-08-rules-data-quality.md` + 对话摘要

## 1. 目标与成功标准

**目标**: 在 ~30-60 分钟内产出"规则与数据质量"的量化报告 + 可执行缺口清单。

**成功标准**:
1. 报告中包含 3 类指标矩阵（规则资产 / 动态测量 / 数据准确度），每项可独立引用
2. 每个缺口附带优先级 P0-P3 和预期影响
3. 报告保存到 `docs/audit/2026-06-08-rules-data-quality.md`
4. **不修改任何业务代码**（只读分析），但允许 git 状态记录

## 2. 审查维度

### 矩阵 1: 规则资产静态分析
- 文件级：`rules/*.yaml` 共 11 个文件的行数、条目数、最后修改时间
- 键级：每个 YAML 的键集合，跨文件冲突检测
- 引用完整性：每个 alias 是否被 `comparator.py` / `rule_cleaner.py` 引用
- 孤儿条目：定义了但从来没用过的 alias
- 重复/冲突：同一 canonical_name 的多个来源
- 关键数字：`aliases.yaml` 总条目数、CF 直接法 49 项 vs 实际 88.93% 匹配率

### 矩阵 2: 动态测量
- 测试套件当前状态（已发现 13 失败 + 1 错误 + 13 跳过）
- 复用 `data/ground_truth_reports/baseline_2019_2022.json` 验证报告的 99.77% / 99.52% / 88.93%
- 复用 47,966 全量基线对比数据
- 分行业匹配率（银行/保险/证券/工业/消费等）
- 分年度匹配率（2000-2010 vs 2011-2022）
- `data/exports_v2/` 与 RDS 真值对比的字段覆盖率
- 2 个未提交导出文件 (sina_cleaned_*.csv) 的具体变更

### 矩阵 3: 数据准确度抽样
- 抽样策略：从基线报告中识别高/低匹配率股票各 2-3 只
- 端到端跑清洗流水线或对比已有的 exports_v2 输出
- 对比 cleaned CSV vs RDS ground truth（值差异、单位错误、缺失项）
- 抽样 3 个常见别名（如"其中：xxx"），人工核对应用结果
- CF 间接法项目当前是否被合理跳过
